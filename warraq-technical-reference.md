### ماذا تبني Lovable؟
Lovable تولّد كود React/TypeScript وتربطه بـ Supabase تلقائياً. الكود الناتج هو ملكك الكاملة ويمكن تصديره في أي وقت.

### تصدير الكود (مهم للطوارئ):
في حال توقفت Lovable أو أردت الاستقلالية الكاملة:
1. افتح مشروعك في Lovable
2. اضغط على **Export to GitHub** أو **Download Code**
3. احتفظ بنسخة من الكود في مكان آمن دورياً

### ماذا يخزن Supabase؟

| الجدول | ماذا يخزن |
|--------|----------|
| `users` / `profiles` | بيانات المستخدمين (اسم، إيميل، تاريخ التسجيل) |
| `items` | العناصر المصنّفة (مهام، أفكار، قرارات) |
| `recordings` | سجل التسجيلات الصوتية أو النصوص المحوّلة |
| `notifications` | الإشعارات لكل مستخدم |
| `subscriptions` | حالة الاشتراك المرتبطة بـ Paddle |
| [يرجى إضافة جداول أخرى] | — |

### مفاتيح API (لا تشاركها أبداً علناً)

| المفتاح | أين تجده | الاستخدام |
|---------|----------|----------|
| `SUPABASE_URL` | Supabase Dashboard → Settings → API | ربط التطبيق بقاعدة البيانات |
| `SUPABASE_ANON_KEY` | Supabase Dashboard → Settings → API | للعمليات العامة (يمكن نشره) |
| `SUPABASE_SERVICE_KEY` | Supabase Dashboard → Settings → API | للعمليات الإدارية (سري جداً ❌) |

> **تحذير**: مفتاح `SERVICE_KEY` يمنح وصولاً كاملاً لقاعدة البيانات. لا تضعه أبداً في كود الواجهة الأمامية.

### النسخ الاحتياطي (Backup)

- Supabase Pro يقدم نسخ احتياطي تلقائي يومي
- يُنصح بتفعيل **Point-in-Time Recovery** من الداشبورد
- [يرجى التعبئة: هل هو مفعّل؟ نعم / لا]

1. سجّل دخول على https://vendors.paddle.com

## ١١. في حالات الطوارئ

### سيناريو: الموقع توقف

```
١. تحقق من Lovable Dashboard — هل هناك مشكلة في النشر؟
٢. تحقق من Supabase Status: https://status.supabase.com
٣. تحقق من Paddle Status: https://www.paddle.com/status  
٤. تواصل مع الدعم التقني الأول: [يرجى التعبئة — الاسم والرقم]
```

### سيناريو: اختُرق حساب

```
١. غيّر كلمة مرور الإيميل الرئيسي فوراً
٢. أوقف مفاتيح Supabase API من الداشبورد (Regenerate Keys)
٣. أوقف مفاتيح Paddle من الداشبورد
٤. أبلّغ Supabase Support: support@supabase.io
٥. راجع جميع المعاملات المالية على Paddle
```

### سيناريو: خسارة الوصول للحسابات

```
لكل خدمة، احتفظ بـ:
- الإيميل الاحتياطي للاسترداد
- رموز الاسترداد (Recovery Codes) للمصادقة الثنائية
- رقم الهاتف المسجل

المسؤول الاحتياطي (Backup Admin): [يرجى التعبئة — اسم وبيانات التواصل]
```
